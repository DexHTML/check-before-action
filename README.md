# check-before-action

SEO-MVP проекта «Проверь перед действием».

## Суть

Статический сайт на GitHub Pages с быстрыми проверками рискованных ситуаций:

- подозрительный звонок;
- просьба о переводе денег;
- ссылка на доставку или оплату;
- код из СМС;
- защита родителей и родственников.

Проект не является официальной юридической, банковской или правоохранительной службой. Цель MVP — проверить SEO-спрос и поведение пользователей без вложений, серверов, PHP, базы данных и платёжных систем.

## Технологии MVP

- HTML
- CSS
- JavaScript
- GitHub Pages

## Структура

- `index.html` — главная страница
- `pages/` — SEO-инструкции
- `tools/` — интерактивные проверки
- `assets/css/` — стили
- `assets/js/` — логика инструментов
- `robots.txt` и `sitemap.xml` — базовая SEO-инфраструктура
- `docs/` — решения, контрольные точки и результаты наблюдений
- `scripts/audit_site.py` — автоматическая проверка HTML, canonical, sitemap и локальных ссылок

## Текущее состояние

- GitHub Pages опубликован: https://dexhtml.github.io/check-before-action/
- В sitemap — 24 целевых URL.
- Подключены Яндекс Метрика, Яндекс Вебмастер и Google Search Console.
- Настроены пять основных целей для продуктовых действий.
- Материалы снабжены canonical, датой редакционной проверки и реестром официальных источников.

## Первый подтверждённый поисковый сигнал

Снимок на 16 июля 2026 года:

- Яндекс Вебмастер: 16 показов, 1 клик, CTR 6,25%;
- Метрика за 10–16 июля: 2 посетителя, 2 визита, 3 просмотра, один поисковый визит;
- первый клик пришёл из кластера проверки подозрительных ссылок и переводов;
- Яндекс показывает в индексе 3 URL проекта;
- Google показывает 2 проиндексированных URL и 7 просканированных, но пока не проиндексированных.

Подробности и ограничения выборки зафиксированы в `docs/SEO_AUDIT_2026-07-16.md`.

## Ближайшее решение

Не расширять сайт массово до накопления выборки. Приоритет — индексация существующих URL, официальные источники, контроль canonical и наблюдение за поисковыми запросами и целями.
